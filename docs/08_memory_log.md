# Memory Log — 永続メモリ＆教訓アーカイブ
> **ECC (Error-Correcting Codex) 思想 v1.0**
> このドキュメントはセッションをまたいで蓄積される「学習記録」である。
> 信頼度スコアが 0.8 を超えた知見は、関連する docs/01-07 へ自動昇格させる。

---

## ECC 思想とは

```
通常のAIセッション: 毎回ゼロから始まる。同じミスを繰り返す。
ECC思想:           過去の失敗を「誤り訂正符号」として記録。
                   確認が取れた知見のみを「真実」として昇格させる。
                   不確実な情報は低スコアで保持し、反証が来たら即廃棄。
```

---

## スコア体系

| スコア | 状態 | 説明 |
|--------|------|------|
| 0.1 – 0.3 | 仮説 | 1回確認。まだ疑わしい |
| 0.4 – 0.6 | 暫定採用 | 複数回確認。ただし例外が存在する可能性 |
| 0.7 – 0.79 | 準確定 | 強い証拠あり。昇格候補 |
| **0.8 – 1.0** | **確定知見** | **docs/01-07 への昇格対象** |
| ~~取消線~~ | 廃棄 | 反証が確認された。参照禁止 |

## 昇格フロー

```
新規教訓を docs/08 に記録（スコア 0.3 から開始）
         ↓
同じ状況で再現確認 → スコア +0.2
         ↓
スコア ≥ 0.8 かつ Gemmaの --mode doc レビューで PASS
         ↓
Orchestrator が該当 docs/XX を開いて知見を追記
docs/08 の当該エントリを「昇格済み」にマーク
```
