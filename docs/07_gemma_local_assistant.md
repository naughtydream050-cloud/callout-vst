# Gemma Local Assistant — 無料ローカルAIの活用定義書
> **System Prompt / Internal Harness v1.0**
> Gemma 2（ローカル動作）を「Windsurf起動前の検閲役」として組み込む。
> Windsurfの無料枠を「正しい作業」にのみ消費させるための前段フィルター。

---

## 0. 設計思想：トークン経済の鉄則

```
【問題】
Windsurfの無料枠 → 有限リソース
ミスった設計図・バグだらけのコードでWindsurfを起動 → 枠を浪費

【解決策】
Gemma（完全無料・ローカル動作・枠なし）を「事前検閲役」に置く。
Windsurfを起動するのは、Gemmaがグリーンライトを出した後のみ。
```

**3層ゲートモデル**:
```
[Human: 設計・アイデア]
       ↓
[Gemma: プレ検閲]  ← タイポ・座標ミス・ロジックエラーを安価に潰す
       ↓（承認後のみ）
[Windsurf: 実装]   ← 有限枠を正しい作業に集中投下
       ↓
[GitHub Actions: CI/CD]
```

## 1. Gemmaにやらせる雑用リスト

### カテゴリA — 設計検証（blueprint確認）
- [ ] JSONスキーマのバリデーション（閉じ括弧・カンマ忘れ）
- [ ] 座標の妥当性チェック（X+W がキャンバス幅を超えていないか等）
- [ ] パラメータIDの重複チェック
- [ ] ノブのW=Hが守られているか（正方形チェック）

### カテゴリB — デバッグアシスト（ビルドエラー初期診断）
- [ ] コンパイルエラーログからタイポ・セミコロン忘れを特定
- [ ] `undefined reference` → ヘッダーのinclude漏れ特定
- [ ] CMakeLists.txtのソースリスト漏れ特定

### カテゴリC — クリエイティブ補助（SNSコピー生成）
- [ ] Instagram/X用の宣伝文10案（Krump魂込み）
- [ ] プラグイン名のネーミング10案（英語・エッジの効いた）
- [ ] リリースノートの日本語→英語翻訳

### カテゴリD — ルーティン確認
- [ ] push_docs_to_github.ps1 の実行前に、対象ファイルリストが正しいか確認
- [ ] GitHub Actions の workflow YAML の構文チェック
- [ ] .gitignore に不要ファイルが漏れていないか確認

## 2. プロンプト集

### ② [Design Check] — blueprint の座標バリデーション

Gemmaに渡すプロンプトテンプレート（`gemma_bridge.py` が自動生成）:

```
あなたはJUCEプラグインのUI設計レビュアーです。
以下のblueprint.mdの内容を、JUCE座標系（原点=左上、X右正、Y下正、単位px）として検証してください。

チェック項目:
1. 各コンポーネントのX + W がキャンバス幅 {canvas_width} を超えていないか
2. 各コンポーネントのY + H がキャンバス高さ {canvas_height} を超えていないか
3. ノブ（RotarySlider）の W = H が守られているか（正方形必須）
4. パラメータIDが重複していないか
5. 全コンポーネントのXYが負の値でないか

---
{blueprint_content}
---

検証結果を以下の形式で返してください:
[PASS] または [FAIL]
失敗した項目: （箇条書き）
修正提案: （具体的な数値で）
```

### ② [Debug Assistant] — ビルドエラー初期診断

```
あなたはC++/JUCEのビルドエラー診断専門家です。
以下のコンパイルエラーログを解析し、「単純なミス（タイポ・セミコロン忘れ・include漏れ）」を特定してください。
複雑なロジックの問題は「要Windsurf対応」として分類してください。

---
{error_log}
---

出力形式:
## 単純なミス（Gemmaが修正案を提示）
- ファイル名:行番号 — 原因 — 修正案

## 複雑な問題（Windsurfに回す）
- 概要: ...

## 判定
[ GEMMA_FIX_POSSIBLE ] または [ NEEDS_WINDSURF ]
```

### ③ [Creative Copy] — Instagram/X 宣伝文 10案（Krump魂）

```
あなたはKrumpダンス文化に精通した音楽プロデューサー向けのコピーライターです。
以下のVSTプラグインのInstagram/X用宣伝文を10案生成してください。

プラグイン情報:
- 名前: {plugin_name}
- 効果: {plugin_description}
- ターゲット: ヒップホップ・Krump系アーティスト

トーンの要件:
- 攻撃的・エネルギッシュ（Krumpのバトル精神）
- 技術的すぎない（感情に訴える）
- ハッシュタグ3〜5個を含む
- 英語または日英ミックスOK
- 各案 140文字以内（X用）

10案を番号付きで出力してください。
```

## 3. Gemmaのセットアップ（ローカル実行）

### 推奨モデル
| モデル | 用途 | VRAM |
|--------|------|------|
| Gemma 2 2B (Q4) | 軽量チェック・SNSコピー | ~3GB |
| Gemma 2 9B (Q4) | コードデバッグ・詳細分析 | ~6GB |
| Gemma 2 27B (Q4) | 複雑なアーキテクチャ判断 | ~16GB |

```bash
ollama pull gemma2:9b
ollama run gemma2:9b

# またはLM Studio経由（GUI）
# https://lmstudio.ai/ からダウンロード
```

### API エンドポイント（Ollama）
```
http://localhost:11434/api/generate
```

## 4. ワークフローへの組み込みルール

### 鉄則: Windsurf を起動する前に必ず Gemma の承認を得ること

```
例外なし。急いでいても、「ちょっとした修正」でも、必ずGemmaを通す。
理由: Windsurfの無料枠は月次リセット。使い切ったら翌月まで待つ必要がある。
```

### Gemmaレビューのタイミング

| タイミング | チェック内容 | 使用プロンプト |
|-----------|------------|--------------|
| Step 1 開始前 | blueprint.md の座標妥当性 | [Design Check] |
| Step 2 開始前 | KnobLookAndFeel の仕様確認 | [Design Check] + ドキュメント照合 |
| Step 3 開始前 | APVTSパラメータIDの重複チェック | [Design Check] |
| ビルドエラー発生時 | エラーログの初期診断 | [Debug Assistant] |
| リリース前 | SNS告知文の準備 | [Creative Copy] |

### Gemma承認チェックリスト（各Step前）

```
[ ] Gemma に blueprint.md を渡した
[ ] [PASS] が返ってきた（または指摘を修正した）
[ ] 指摘事項をすべてblueprintに反映した
[ ] 再チェックで [PASS] を確認した
→ ここまで完了して初めてWindsurfに指示を出す
```

## 5. Gemma の限界と Windsurf へのエスカレーション基準

Gemma は以下を **判断できない**。これらはすべて Windsurf に任せること。

| 問題タイプ | 理由 | エスカレーション先 |
|-----------|------|-----------------|
| JUCEの実際の描画バグ | レンダリング結果を見られない | Windsurf |
| 複雑なDSPアルゴリズム | 音響工学的な判断が必要 | Windsurf |
| CMakeのプラットフォーム固有問題 | 実行環境情報が必要 | Jules / Windsurf |
| パフォーマンス最適化 | プロファイル結果が必要 | Windsurf |
